JSON (Java Script Object Notation)
- довольно простая грамматика (30 паттернов)
- парсится недолго
- позволяет произвольно форматировать вывод пробелами и переносами строк
– облегченный формат обмена данными между компьютерами
- программирование web-приложений, где он служит альтернативой XML
- подходит для сериализации сложных структур

XML (Extensible Markup Language)
- довольно сложная грамматика (90 паттернов)
- парсится по времени в среднем между json и yaml
- позволяет произвольно форматировать вывод пробелами и переносами строк
- позволяет определять собственные теги и атрибуты
- универсальность

YAML  («Ain’t Markup Language»)
- По данным Википедии - синтаксис YAML минималистичен, особенно по сравнению с XML синтаксисом. По другим данным - имеет крайне сложную грамматику (210 паттернов).
- парсится на порядок дольше из-за сложности синтаксиса
- человекочитаемость
- в основном используется как формат для файлов конфигурации
