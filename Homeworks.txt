2.1. Занятие «Открытие и чтение файла, запись в файл»
1. Список рецептов хранится в формате:
Название блюда
Kоличество ингредиентов
Название ингредиента | Количество | Единица измерения
Пример:

Омлет
3
Яйца | 2 | шт
Молоко | 50 | г
Помидор | 100 | мл

2. Блюд может быть больше одного в одном файле;
3. Читать список рецептов из файла;
4. Залить результат на github и прислать ссылку;
5. Написать для чего используются типы данных: json, xml, yaml.
Програма должна читать файл cook_book.txt и использовать полученные данные в формате словаря для формирования списка покупок на основании дополнительно введенных данных о ноиенклатуре и количестве блюд.
Т.е. в файле shop_list.py переменная cook_book должна формироваться в формате словаря в результате чтения списка рецептов из файла cook_book.txt.
– Данный функционал нужно интегрировать в код, который был написан на лекции. Т.е. итоговая программа должна читать список блюд из файла, запрашивать нужные блюда у пользователя и выводить список покупок.
– Нужно, чтобы количество ингредиентов складывалось по наименованию.
– Обратить внимание на строки 37-38. return всегда прекращает работу функции, поэтому print отрабатывать никогда не будет.
– Будет лучше если список блюд будет отображаться для пользователя в отформатированном виде, а не словарем, чтобы было удобнее выбирать (речь идет о попытке вывести словарь на строке 37).
Реализовано: https://github.com/pashkovsky/PY2PM/blob/master/shop_list.py
